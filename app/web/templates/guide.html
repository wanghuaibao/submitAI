使用指南

快速入门

通过以下四个简单步骤开始使用我们的自动化产品提交工具：

1

填写产品信息

在"创建新提交"页面填写您的AI工具详细信息，包括名称、URL、描述等。确保提供准确信息并上传清晰的产品Logo。

2

选择目标目录

勾选您希望将产品提交到的目录网站。您可以同时选择多个目录，系统将自动处理所有提交。

3

开始自动提交

提交表单后，您将被重定向到提交列表页面。点击相应提交的"查看"按钮，然后点击"开始提交"启动自动提交流程。

4

查看提交结果

系统将在后台处理提交，并实时更新状态。您可以随时刷新页面查看最新进度，包括成功状态和错误信息（如有）。

测试指南

如果您想测试系统功能，请按照以下步骤操作：

1. 启动应用

使用以下命令启动应用：

# MacOS/Linux
./start.sh

# Windows
start.bat

确保应用成功启动，并显示"应用已在 http://localhost:8000 启动"消息。

2. 访问Web界面

在浏览器中打开以下地址：

http://localhost:8000

确保能看到首页内容，并且各个链接正常工作。

3. 创建测试提交

点击"创建新提交"按钮，使用以下测试数据：

  • 产品名称：Test AI Tool
  • 产品网址：https://example.com
  • 简短描述：这是一个用于测试的AI工具
  • 详细描述：这是一个测试描述，用于验证系统功能是否正常工作。包含详细信息和特点说明。
  • 产品分类：选择任意一个
  • 价格模型：选择任意一个
  • 标签：test,ai,demo
  • 上传测试Logo：使用任何图片文件

4. 启动测试提交

在提交列表页面，找到您刚创建的测试提交，点击"查看"，然后点击"开始提交"按钮。

系统将在后台启动浏览器（无头模式），并开始自动提交流程。

5. 确认测试成功

测试成功的标准：

  • 状态变为"已完成"
  • 各目标网站显示"提交成功"状态
  • logs目录中包含表单填写和提交过程的截图
  • logs/submissions目录包含提交记录和结果

注：测试提交时不会实际提交到目录网站，但会模拟完整流程。

常见问题解答

应用无法启动怎么办？

确认是否已安装所有依赖，可以尝试手动运行以下命令：
pip install -r requirements.txt
playwright install chromium

提交过程卡在"进行中"状态？

检查logs/submitter_日期.log文件查看详细错误信息。可能的原因包括网络问题、目录网站暂时不可用或表单结构变化。

如何查看提交的详细过程？

在logs/submissions目录下查找以您提交ID开头的文件，包括表单截图、结果截图和表单字段映射信息。

如何在公共服务器上部署？

可以使用Docker或直接在支持Python的服务器上部署。确保服务器上已安装Python 3.8+、必要依赖和Playwright。
完整部署指南请参考部署文档。

提交失败的常见原因？

  • 产品描述过短或包含不适当内容
  • 网络链接不可访问
  • 产品Logo分辨率过低或尺寸不合适
  • 目录网站表单结构变化
  • 目录网站暂时维护或不可用

提交后多久能在目录网站上看到？

大多数目录网站需要人工审核，通常需要1-7天的处理时间。成功提交不代表一定会被发布，内容仍需符合各网站规范。

如果您有其他问题或需要帮助，请联系我们的支持团队。
